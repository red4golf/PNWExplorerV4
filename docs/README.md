# Documentation Structure

Project documentation has been grouped by function to reduce root-level clutter and align with common repository conventions:

- `docs/operations/`: deployment, infrastructure, and analytics runbooks.
- `docs/book-verification/`: book verification system references and tracking artifacts.
- `docs/marketing/`: launch, newsletter, and announcement materials.
- `docs/qa/`: quality-review checklists and review notes.

When adding new documentation, place it in the closest functional folder and avoid adding one-off markdown files at the repository root.
